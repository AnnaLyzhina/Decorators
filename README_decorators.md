# Decorators

Домашнее задание к лекции **«Decorators»**.

В репозитории реализованы 3 задания:

1. `logger` — декоратор без параметров, который пишет логи в `main.log`;
2. `logger(path)` — параметризованный декоратор, который пишет логи в указанный файл;
3. применение декоратора к предыдущему проекту — Telegram-боту **TG-API**.

## Структура проекта

```text
Decorators/
├── 1.logger/
│   └── main.py
├── 2.logger_with_path/
│   └── main.py
├── 3.application_logger/
│   ├── src/
│   │   ├── bot.py
│   │   ├── db.py
│   │   ├── repository.py
│   │   └── logger.py
│   ├── requirements.txt
│   └── README.md
└── README.md
```

## Задание 1

Реализован декоратор `logger`, который записывает в файл:

- дату и время вызова функции;
- имя функции;
- аргументы `args` и `kwargs`;
- возвращаемое значение.

Лог записывается в файл `main.log`.

## Задание 2

Реализован параметризованный декоратор `logger(path)`, который делает то же самое, но путь к лог-файлу передаётся в аргументе декоратора.

Пример:

```python
@logger('log_1.log')
def hello_world():
    return 'Hello World'
```

## Задание 3

Для 3 задания логгер применён к приложению из прошлой домашней работы — проекту **TG-API**.

Идея применения:
- логировать вызовы функций работы с базой данных;
- логировать ключевые пользовательские действия в боте;
- при необходимости логировать ошибки и результаты обработки команд.

Подходящие функции для декорирования:
- методы из `repository.py`;
- функции обработки команд из `bot.py`;
- вспомогательные функции, связанные с выборкой слов, добавлением слов и записью истории ответов.
